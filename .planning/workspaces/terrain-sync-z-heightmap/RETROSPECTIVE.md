# Retrospective

## Planned vs Actual
- Where did execution match or diverge from the plan?

## What Worked
- Which decisions or tools helped?

## What Hurt
- Which assumptions, tools, or gaps slowed the work down?

## Principle Updates
- Which principles should be added, changed, or retired?

## Tooling Updates
- Which tools should be adopted, updated, or created?

## Debugging Improvements
- Which debug-loop tactics, audit heuristics, or retained inspection hooks should improve future work?

## Summary Updates
- Which findings should be pulled into `PROJECT_SUMMARY.md`?

## Next-Time Changes
- What should be done differently on the next similar task?
