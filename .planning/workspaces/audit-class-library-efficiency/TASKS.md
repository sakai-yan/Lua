# Tasks

## Active
- Work currently being executed
- Keep the default continuation target as the first actionable item here.
- Do not leave clearly in-scope, currently feasible implementation work here as outline-only progress.
- When debug-loop work is active, keep the current debug continuation target as the first actionable item here.

## Ready
- Work that is unblocked and can start next
- Order items by next execution priority; the first actionable item is the fallback continuation target.
- Move the next debug experiment, verification step, or audit follow-up here when it is not yet active.

## Blocked
- Work waiting on decisions, dependencies, or new information
- Move stale or non-executable items here instead of letting them clutter `Active`.
- Move unresolved reproduction or missing-observation prerequisites here when debug progress is blocked.

## Done
- Finished work with validation notes or references
